图解http

1.了解web以及网络基础

​ 1.tcp/ip 分层：1.应用层 http 2.传输层 tcp 3. 网络层 ip 4.链路层 以太网

​ 客户端 经过4次封装 –> 服务端 经过4 解析 获取客户的请求

​ 2.与http密切相关的协议:tcp,ip,dns

​ ip 属于网络层，ip协议用来把各种数据包传送给对方，ip地址指明了节点被分配到的地址， mac地址是指网卡所属的固定地址

​ ip地址可以和mac地址进行配对

​ 使用arp协议凭借mac地址进行通信，通信双方是通过多台计算机和网络设备中转才连接到对方，在中转时，会利用下一站的mac 地址找到下一个中转目标

​ tcp 属于传输层，用来提供可靠的字节流服务。确认数据是否送达对方（常见的是 三次握手

​ dns属于应用层，提供域名到ip地址间的解析服务。

​ 3.uri 和 url url是uri（标志符的子集

2.简单的http协议

​ 1.http协议用于客户端和服务端之间的通信

​ 请求报文是由请求方法，请求uri ，协议版本，可选的请求首部字段和内容实体构成的。

​ 响应报文是由协议版本，状态码，状态码原因短语，响应首部字段和实体构成。

​ 2.http是不保存状态的协议，即无状态协议

​ 但是这样会导致业务处理比较困难，比如购物时，添加购物车，所以这里为了解决这个问题引入了Cookie技术，来实现期望的保持状态功能。

​ 无状态协议的优点：减少服务器的cpu以及内存资源的消耗

​ cookie：是通过在请求和响应报文中写入Cookie来控制客户端的状态

​ 有服务器生成cookie并返回给客户端

3.http报文内的http信息

​ 1.http报文：a.请求行，请求首部字段，通用首部字段，实体首部字段，其他

​ b.状态行，响应首部字段，通用首部字段，实体首部字段，其他

​ 2.编码提升传输熟虑

​ 3.发送多种数据的多部分对象集合

​ 4.内容协商返回合适内容（即浏览器默认语言是什么就返回什么语言（通过服务器驱动，客户端驱动，透明协商来判断

4.返回结果的http状态码

  1. 1XX 表示接受的请求正在处理
  2. 2XX 表示请求正常处理完毕
  3. 3XX 表示需要附加操作来完成请求
  4. 4xx 表示服务器无法处理请求（客户端请求有问题
  5. 5XX 表示服务器请求出错

5.与http协作的web服务器

  1. 用单台虚拟机实现多个域名：我们在访问网站时，域名会通过DNS服务映射到ip地址之后访问目标网站，所以当有多个域名时，要在host首部内完整置顶主机名或域名uri；

  2. 通信数据转发程序：代理，网关，隧道
    代理：是一种由转发功能的应用程序，是服务器 和客户端的中间站

    使用代理服务器的优点：利用缓存技术减少网络带宽的流量，组织内部针对特定网站的访问控制，以获取访问日志为主要目的。（google

    代理可分为两种：1.是否利用缓存2.是否修改报文（不修改为透明代理，修改为非透明代理

    网关: 是转发其他服务器通信数据的服务器

    能提供非http协议服务，能提高通信的安全性

    隧道：是在相隔甚远的客户端和服务器两者之前进行中转，并保持双方通信连接的应用程序

    建立一条通信路线，使用ssl等加密手段进行通信。

  3. 保存资源的缓存（服务器端的缓存有效期限，客户端的缓存

6.http首部

  1. http报文首部（见3.1
    1. 请求行，请求首部字段，通用首部字段，实体首部字段，其他
    2. 使用首部字段是为了给浏览器和服务器提供报文主体大小，所使用的语言，认证信息等内容
    3. 出现重复的http首部字段，那么会因为浏览器的不同而出现不同的处理结果。

7.确保web安全的https

8.确认访问用户身份的认证

9.基于http的功能追加协议

  1. http存在性能瓶颈
  2. google发布SPDY的缩短了50%的加载时间（瓶颈a.一个连接一个请求 b.请求只能从客户端开始 c.请求和响应首部未经压缩就发送，首部信息越多延迟越大 d.发送冗长的首部，每次互相发送相同的首部造成的浪费较大 e.可以选择任意数据压缩格式，非强制压缩
  3. Ajax （异步 js xml) 达到局部刷新的目的，核心技术是（XML HttpRequest）
  4. Comet （先响应客户端请求，保持连接直到内容更新）
  5. 使用WebSocket，web浏览器和web服务器之间全双工通信标准

一旦双方建立websocket协议通信，任意一方都可以直接向对方发送报文。（那么之前的通信协议会经过哪几个步骤通信呢）

​ websocket协议特点（1.服务器推送功能 2.减少通信量）

10.构建web内容的技术

  1. HTML (DOM:用来操作html和xml)
  2. 与web服务器及程序协作的CGI(cgi会在每次收到请求后都启动一下，大访问量下消耗资源略大)
  3. java Servlet （servlet常驻内存，运行在web容器)
  4. XML可扩展标记语言
  5. JSON轻量级数据标记语言

11.web的攻击技术

  1. 攻击主要是（web服务 ，sql注入，os注入）
  2. 输出值转义不完全（跨站脚本攻击（xxs)，sql注入，）
  3. 设置或者设计缺陷
  4. 会话管理疏忽引发的安全漏洞
